# Horizon Worlds 3P Snackables Content

Repository for third-party Meta Horizon World submissions. Each world is stored as an **independent branch** — your branch contains your complete MHS project and does not interfere with other submissions.

**Repository**: `https://github.com/facebookexternal/Horizon-Worlds-3P-Snackables-Content`

## Branch Naming Convention

```
<studioname>-<world-name>
```

Use lowercase, examples:
- `bigboxvr-hole-io`
- `resolution-bopit`
- `studioalpha-tower-defense`

---

## Prerequisites

- [Git](https://git-scm.com/downloads) installed on your machine
- GitHub access to the `facebookexternal/Horizon-Worlds-3P-Snackables-Content` repository
- Your completed MHS project folder

---

## Step-by-Step Submission

### 1. Open a terminal in your project folder

Navigate to your MHS project root (the folder containing your `.hzproject` file):

```powershell
cd "C:\path\to\your\YourProject"
```

### 2. Initialize Git and connect to the repository

```powershell
git init
git remote add origin https://github.com/facebookexternal/Horizon-Worlds-3P-Snackables-Content.git
```

### 3. Create your branch (forked from main)
```powershell
git fetch origin main
git checkout -b <studio_name>-<world_name> origin/main
```
For example:
```powershell
git fetch origin main
git checkout -b bigboxvr-hole_io origin/main
```
> **Important:** Your branch must be forked from `main` so that it inherits the CI checks (`.github/` folder). Do **not** use `--orphan` to create your branch.

### 4. Add a `.gitignore` file

Create a `.gitignore` file in your project root with the following content to exclude auto-generated files:

```
node_modules/
tsconfig.json
.vscode/
```

> `tsconfig.json` is auto-generated by Meta Horizon Studio and should not be committed. Use `tsconfig.custom.json` for any custom TypeScript settings.

### 5. Stage and commit your files

```powershell
git add .
git commit -m "<WorldName>: initial submission"
```

### 6. Push your branch

```powershell
git push -u origin <studio_name>-<world_name>
```

---

## Updating Your Submission

After your initial push, to submit updates:

```powershell
cd "C:\path\to\your\YourProject"
git add .
git commit -m "Brief description of changes"
git push
```

## Troubleshooting

**"remote already exists"** — If you've previously initialized git in this folder:

```powershell
git remote set-url origin https://github.com/facebookexternal/Horizon-Worlds-3P-Snackables-Content.git
```

**"failed to push"** — Ensure you have write access to the repository. Contact the repository admin if needed.

**"branch already exists on remote"** — Your studio/world branch name may already be taken. Verify with:

```powershell
git ls-remote --heads origin
```
